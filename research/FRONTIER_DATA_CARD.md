# Frontier Fixture Data Card

## Summary

The paired frontier fixture contains 180 synthetic support-task IDs evaluated under
four deterministic agent configuration arms. It is generated by
[`examples/compute-frontier/generate.py`](../examples/compute-frontier/generate.py).

## Purpose

The fixture tests exact pairing, full-cost reconstruction, harmful-regression bounds,
paired cost intervals, multiplicity adjustment, assurance gating, Pareto labeling,
and deterministic report generation.

It is not intended to estimate production agent performance or economic impact.

## Records

Each arm contains:

- 180 model events with an explicit direct cost;
- 180 binary outcomes with explicit business value, human minutes, remediation cost,
  and incident loss;
- one shared baseline and economic policy; and
- route, step budget, arm ID, and prompt version metadata in event arguments.

The separate `task-manifest.json` contains 180 shared SHA-256 input digests and one
shared rubric version. Its digest is frozen in the experiment plan and injected into
each normalized evidence bundle. Task identities are therefore identical across
arms while outcomes and costs differ by controlled arm.

## Generation

The generator uses fixed index sets rather than randomness. Re-running it produces
byte-identical normalized JSON files. Frontier bootstrap randomness is separate and
uses the seed frozen in `manifest.json`.

## Known limitations

- No prompts, responses, tools, or real model calls are present.
- The outcome rubric is simulated and has no annotator agreement measurement.
- Failure patterns are constructed rather than sampled from a workload.
- Cost fields illustrate full-cost accounting but are not market estimates.
- Task independence is assumed by the statistical procedure and manufactured by the
  fixture, not empirically validated.
