## Launch sequence

1. Publish and verify the flagship repository from a clean checkout.
2. Create the profile README and pin only defensible artifacts.
3. Publish the technical launch post with the reproduction command.
4. Publish a narrow methodological follow-up 48–72 hours later.
5. Invite one concrete contribution type per post.
6. Track issues, reproduction failures, counterexamples, and adapter requests.

## Primary post

1. Open with the hard question.
2. Name what existing stacks already do well.
3. State the missing decision boundary.
4. Show the add/delete or falsification result.
5. Include one limitation.
6. Provide one command and one link.
7. Ask for one concrete contribution.

## Success measures

- successful independent reproductions;
- high-quality issues or claim challenges;
- contributed fixtures, checks, or cases;
- external citations or technical discussion;
- qualified collaboration or advisory conversations; and
- fixes driven by counterexamples.

Do not optimize only for views, likes, stars, or email capture.
